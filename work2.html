ECOALA

Portfolio

팀 프로젝트를 시작하기 앞서 Figma 툴을 활용하여 화면들의 구성을 제작하였습니다. 피그마로 페이지의 구성 및 디자인을 제가 담당했습니다.

ECOALA는 우리 집의 전기 사용량을 분석해주는 대쉬보드형 페이지입니다. 처음 접속하면 나오는 화면이며, 사용자 로그인, 관리자 로그인이 있습니다. 사용자와 관리자를 분리하여 각자 로그인 할 시 볼 수 있는 시스템이 다릅니다. 우측 상단에는 '회원가입'이 있고 클릭할 시 회원가입 페이지로 이동합니다. 페이지의 디자인과 제작을 제가 담당했습니다.

유저의 로그인 화면입니다. 유저는 회원가입을 한 후 자신의 대쉬보드로 이동해 서비스를 제공받을 수 있습니다. 아이디 기억하기 기능이 추가되어 있습니다. 유저 로그인 페이지의 디자인과 프론트엔드 제작을 제가 담당했습니다.

관리자 로그인 화면입니다. 관리자는 회원가입 필요없이 저장된 아이디와 비밀번호로 접속 할 수 있습니다. 관리자 로그인 페이지의 디자인과 프론트엔드 제작을 제가 담당했습니다.

회원가입 페이지입니다. 저희 'Ecoala'는 동일 조건에 있는 사용자의 데이터를 비교하는 기능이 있기 때문에 필수사항들을 입력받고 있습니다. 회원가입 페이지의 디자인과 프론트엔드 제작을 제가 담당했습니다.

저희 'Ecoala' 각 팀원마다 한 페이지의 풀스택 경험을 원했기 때문에 제가 초기의 프론트엔드 틀을 제작해서 팀원들에게 'svn'을 통해 배포했습니다. 이 화면은 팀원들에게 배포한 메인페이지의 기본 틀입니다.

'Ecoala'의 메인 페이지입니다. 좌측 상단에는 오늘의 전력 사용량이 업데이트됩니다. 우측 상단에는 사용한 전력 사용량에 대한 전기 요금이 계산되어 나오게 됩니다. 계산식은 한국전력의 계산식을 사용해서 계산했습니다. 좌측 하단에는 가전 별 전력 사용량이 업데이트 됩니다. 예로 냉장고, 에어컨, TV 등 전력량이 많은 순서대로 5개가 원형 그래프로 표시됩니다. 우측 하단에는 탄소 배출 계산기 현재 사용한 전력에 비례 이산화탄소 발생량과, 이를 사라지게 하려면 얼마나 나무가 필요한지를 계산해 사용자들에게 심각성을 알립니다.

탄소 포인트 화면입니다. 좌측에는 탄소 포인트 부여 기준에 대해서 사용자가 알기 쉽게 표기해두었습니다. 우측 상단에는 해당 로그인 유저의 포인트가 나옵니다. 지급 받은 포인트를 통해 우측 하단에 있는 상품으로 교환할 수 있습니다. 이 페이지는 프론트엔드 백엔드를 제가 담당하였으며, AJAX 와 JS, JAVA 를 통해 백엔드를 구성하였습니다.

상품 교환의 버튼을 누른 모습입니다. 해당 유저의 포인트가 충분하면 상품으로 교환이 가능하게 만들었습니다. 또한 사용한 포인트 만큼 포인트가 차감되게 됩니다. 만약 화면처럼 포인트가 부족하다면 화면에 '포인트가 부족하여 교환이 불가합니다'라는 안내 메세지를 표출합니다.
